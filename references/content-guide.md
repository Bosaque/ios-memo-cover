# 内容写作指南

## 小红书备忘录风格文案要点

### 核心原则
- 像真人随手记录，不像精心设计的广告
- 有具体数字（房租/时间/价格），可信度高
- 口语化断句，每行不超过18字
- 结尾留互动引导（你们怎么看？评论区说说）

### 推荐结构
```
开头：一句话点明主题（1-2行）
中间：分3-4个小块，每块3-4行，用None空行分隔
结尾：互动引导（1-2行灰色小字）
```

### 字体颜色搭配
- 标题类：`'body'` + `'dark'`（黑色正文）
- 分类标题（🏠租金）：`'body'` + `'dark'`
- 详情说明：`'body'` + `'dark'`
- 踩坑/补充：`'sm'` + `'sec'`（深灰小字）
- 互动引导：`'sm'` + `'mid'`（中灰小字）

### 好的内容示例
```python
TITLE = '福州租房避坑记录'
lines = [
    ('在福州租房3年，搬了3次家', 'body', 'dark'),
    ('说说真实感受，不吹不黑 👇', 'body', 'dark'),
    (None, None, None),
    ('🏠 租金', 'body', 'dark'),
    ('茶亭1700 vs 白湖亭1200', 'body', 'dark'),
    ('差了500，压力少了好多', 'body', 'dark'),
    (None, None, None),
    ('踩坑总结 👇', 'sm', 'sec'),
    ('1. 签合同前问清水电标准', 'sm', 'sec'),
    ('2. 别信"市场价"，多对比', 'sm', 'sec'),
    (None, None, None),
    ('你们在福州住哪个片区？', 'body', 'dark'),
    ('租金多少，评论区说说 😊', 'sm', 'mid'),
]
```

### 避免的写法
- ❌ 首先/其次/综上所述（AI味）
- ❌ 每行超过18字（显示不全）
- ❌ 内容少于15行（画面空旷）
- ❌ 没有互动引导（降低评论率）
